# Версии превью YurClick

Каждая правка деплоится на отдельный URL. Это позволяет переключаться между версиями и сравнивать их визуально.

## Соглашение

- Формат версии: `vX.Y.Z` (мажор `v5` = новая ветка развития)
- Каждая новая версия = отдельный коммит + отдельный URL деплоя
- Активной считается **последняя** в списке
- Старые версии остаются доступными по своим URL

## Текущая активная версия

**v5.0.0** — Старт мажорной версии v5. Новая страница «Настройки» с двумя вкладками-underline: «Интеграции» (карточки Битрикс24 «Подключено» + AmoCRM «Не подключено» с полями URL/webhook/отдел/стадия и тогглами синхронизации) и «Брендинг» (весь контент бывшей страницы branding.html: домен, цвета, логотип, настройки платформы). Новая страница «Чаты с клиентами» — двухпанельный мессенджер: слева поиск + список 10 диалогов (аватар, ФИО, № дела ссылкой, превью с префиксом «Вы:», время, бейдж непрочитанных indigo), справа лента с группировкой по дням (Сегодня/Вчера/dd month), бабблы own indigo / client white, индикатор «прочитано» (одиночная/двойная галочка), composer (textarea авто-рост + кнопка скрепки + кнопка отправки, Enter — отправить, Shift+Enter — перенос). Поддержка `?clientId=N` авто-открывает диалог и сбрасывает unread. В sidebar убран пункт «Брендинг», добавлены «Чаты» (под «Клиенты») и «Настройки» (внизу). На странице «Клиенты» кнопка «Написать»/«Ответить» теперь `<a href="chats.html?clientId=${idx}">`. Файл branding.html удалён.

## Предыдущая версия

**v4.11.24** — Удалён столбец «Регион» (grid → 7 колонок); у строки клиента убран rounded-sm с card-main; при выделении 2+ клиентов на вкладке «Подготовка документов» появляется sticky bulk-bar с кнопками «Снять выделение» и «Подать в суд» (спиннер + переход в court + toast). Чекбоксы на court-вкладке disabled. Удалён весь код region-popover.

## Как откатиться к предыдущей версии

Сообщить агенту: «откати к v5.X.Y» или «откати к v4.11.X». Текущее состояние перезапишется состоянием указанной версии.

## Как сравнить две версии

Открыть оба URL в соседних вкладках браузера. URL остаются рабочими бессрочно.
